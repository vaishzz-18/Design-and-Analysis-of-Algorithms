Algorithms and Data Structures Overview

Detailed explanation of algorithms and data structures used in the project.

1. Arrays

Functions: write_into_file, read_from_file, bubble_sort_distance, selection_sort_fare, insertion_sort_seats, display_sorted_list.

Purpose: Arrays are used to store details about routes, vehicles, costs, and other data in an indexed format for easy access and sorting.

2. Structures

Functions: Structs like CP, NODE, Resort, and PAYMENT define custom data types for various components (routes, pollution levels, resorts, payments).

Purpose: Structures provide a way to group related data for operations like route management, pollution tracking, etc.

3. Linked List

Functions: create_node, insert_end, display_list, search_vehicle_pollution.

Purpose: Used to store pollution levels and associated vehicle types dynamically.

4. Stack

Functions: push, pop_cost_of_vehicle, display_stack, quick_sort.

Purpose: Implements payment management by storing payment modes and costs for vehicles.

5. Queue

Functions: initialize_queue, enqueue, display_queue.

Purpose: Handles feedback collection and displays user inputs sequentially.

6. Binary Search Tree

Functions: insert_resort, search_resort_by_rating, inorder_traversal.

Purpose: Stores resorts and allows searching based on ratings using in-order traversal.

7. Fenwick Tree

Functions: update, query.

Purpose: Used for managing installment payments efficiently with range queries.

8. Graph Algorithms (DFS, BFS, Dijkstra, Prim, Kruskal)

Functions: DFS, bfs, dijkstra, prim_mst, kruskals.

Purpose: Perform graph traversals and find shortest paths or minimum spanning trees in the city map.

9. Sorting Algorithms (Bubble Sort, Selection Sort, Insertion Sort, Heap Sort, Quick Sort)

Functions: bubble_sort_distance, selection_sort_fare, insertion_sort_seats, heap_sort, quick_sort.

Purpose: Sort routes or vehicles based on cost, distance, or number of seats.

10. Floyd-Warshall Algorithm

Functions: floyd_warshall.

Purpose: Finds shortest paths between all pairs of vertices in the graph.

11. Rabin-Karp String Matching

Functions: rabin_karp.

Purpose: Searches for specific strings (e.g., pollution-related data).

12. Lookup Table

Functions: look_up_table, data_look_up.

Purpose: Precomputes time/distance between city locations for quick retrieval.
